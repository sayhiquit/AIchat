# AI人际管家项目进度

更新时间：2026-07-21

## 1. 当前定位

AI人际管家已经从想法原型进入本地 MVP 软件阶段。目标是帮助用户管理人物关系、真实记忆、沟通风格、汇报节奏和回复策略，让用户在现实沟通中更有准备。

当前主线：桌面软件版。网页文件仍用于 Electron 界面加载，但不再把网页版作为单独测试主线。

## 2. 已完成能力

### 人际关系地图

- 默认以用户为中心展示关系脉络。
- 支持分类分支展开、收缩、聚焦、回到中心。
- 支持点击人物后右侧展开详情。
- 左侧工作区可收起，适配小屏窗口。

### 管理与画像

- 分类和人物支持新增、编辑、删除。
- 人物支持背景、重要程度、亲近程度、联系频率、性格、注意事项、关系策略等字段。
- 管理页新增人物搜索、重要程度筛选和排序。
- 人物详情支持历史记忆编辑、删除和聊天记录导入。

### 对话助手

- 使用聊天软件式布局，左侧历史对话，中间对话，右侧分析。
- 支持对方开场和我方开场。
- 我方消息可先显示，对方生成时展示“正在输入中”气泡。
- 支持 A/B/C 推荐、自定义发送、修改 AI 预测的对方回复后继续生成建议。
- 模拟对话默认不写入真实历史，只有点击新增记忆并保存后才进入长期记忆。

### 蒸馏与导入

- 支持手动粘贴聊天片段。
- 支持导入 TXT、CSV、JSON、Markdown、HTML，并尝试识别 BAK/BAT。
- 导入人物聊天记录时先展示预览、角色统计和样例，确认后才蒸馏。
- 蒸馏提示词已区分稳定模式、单次情绪、未知说话人和人物背景。

### 日程

- 日程页从展示型待办升级为时间表。
- 支持自动关系维护建议。
- 支持手动新增日程、绑定人物、完成、延期、忽略和删除。

### 设置与数据

- 设置页拆分为默认关系、模型、记忆说明、记忆迁移和诊断模块。
- AI 模块支持 DeepSeek 预设、OpenAI 兼容接口、Base URL、模型名、API Key 和连接测试。
- 记忆迁移支持普通 JSON 导出导入和加密备份。
- 诊断模块展示版本、人物数、分类数、记忆数、AI 状态、存储位置和最近备份。

### 桌面版工程

- Electron 桌面版已可运行和打包。
- 已配置最小窗口、自适应布局和自动更新入口。
- 安装包更新不会覆盖用户本地数据。
- 已建立 `npm.cmd run validate` 验证脚本。

## 3. 当前进度判断

- 本地 MVP 完成度：约 82%
- 本地真实使用完成度：约 65%
- 可分发桌面版完成度：约 58%
- 可公开上线产品完成度：约 32%

现在已经适合自己长期试用和小范围发安装包测试，但还不适合公开上线。

## 4. 主要短板

- AI Key 仍在本地前端配置，公开上线前需要后端代理。
- 没有账号系统、云端同步和多设备冲突处理。
- 聊天记录导入还需要针对微信、QQ、钉钉、飞书做更稳定的格式适配。
- 目前没有自动化 UI 端到端测试，仍需要人工回归。
- 自动更新需要服务器和更新源后才能真实启用。

## 5. 下一阶段建议

1. 真实使用 1-2 周，记录高频卡点。
2. 优先补强聊天记录导入格式和人物画像字段。
3. 做后端模型代理，避免正式上线时暴露 API Key。
4. 再做账号、云同步和自动更新源。
5. 最后再考虑聊天软件悬浮助手、手机端和声纹记录等长期能力。

## 6. 当前结论

项目已经具备“可真实试用的软件雏形”。现在最重要的是稳定流程、积累真实样本、修导入体验，并准备后端代理和云同步方案。
