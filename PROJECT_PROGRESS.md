# AI人际管家项目进度

更新时间：2026-07-17

## 1. 项目定位

AI人际管家是一个本地优先的人际关系处理助手。目标是帮助用户管理不同关系中的人物画像、历史记忆、沟通风格、汇报策略和回复建议，让用户在现实对话中更从容。

当前项目形态：

- 网页版：功能测试版，用来快速迭代。
- 桌面版：真实使用版，基于 Electron 加载同一套页面。
- 数据存储：当前仍以本地缓存为主，支持导出和导入记忆。
- AI 接入：已经提供在线模型配置入口，支持 DeepSeek / OpenAI 兼容接口；正式产品仍建议通过后端代理调用模型。

## 2. 已完成能力

### 2.1 人际关系地图

- 默认以“我”为中心展示关系脉络。
- 支持同事、朋友、亲人、客户、上一家公司、同学等默认分类。
- 点击分类后，地图视角移动到该关系分支。
- 点击人物后，人物聚焦并在右侧展开详情。
- 切换分支时旧分支会收缩和退场，减少残留感。
- 左侧关系事项面板支持收起，适配小屏。

### 2.2 分类和人物管理

- 支持分类新增、编辑、删除。
- 支持人物新增、编辑、删除。
- 支持清空预设人物和恢复默认模板。
- 删除人物有二次确认。
- 人物字段包含关系、重要程度、性格画像、注意事项、人物背景、关系分量、亲近程度、联系频率。
- 已补充状态规范化，导入坏数据或删除人物后会自动修正选中状态。

### 2.3 人物画像与记忆蒸馏

- 人物详情页展示说话风格、做事风格、好感触发、避雷点、推荐回复结构、关系策略。
- 支持手动粘贴聊天片段进行画像蒸馏。
- 支持导入 TXT、CSV、JSON、Markdown、HTML，并尝试识别 BAK/BAT。
- 支持历史记忆编辑和删除。
- 普通模拟不会自动写入长期记忆，只有点击新增记忆并保存后才会沉淀。

### 2.4 对话助手

- 使用接近 QQ / 微信聊天框的结构。
- 支持“对方所说”和“我方述说”两种模式。
- 支持 A/B/C 推荐回复和自定义回复。
- 支持自动预测对方可能回复。
- 支持修改 AI 预测的对方回复，再重新生成我方建议。
- 支持右侧整段对话分析和策略咨询。
- 支持“正在输入中”动态气泡和消息弹出动画。

### 2.5 AI 配置

- 设置页提供模型模块。
- 支持 DeepSeek 预设。
- 支持 Base URL、模型名、API Key 配置。
- 支持测试连接。
- 请求失败时会回退本地规则模拟。

### 2.6 数据迁移和离线模式

- 支持导出本地记忆 JSON。
- 支持导入记忆 JSON。
- 设置页显示当前存储位置。
- 保留离线模式。
- 桌面版安装包更新不会覆盖用户本地数据。

### 2.7 日程页

- 已从普通待办列表升级为时间表。
- 展示今日关系维护节奏、月历、本周维护分布和复盘点。
- 人物相关日程可跳转到人物详情。

## 4. 主要短板

### 4.1 AI 调用仍应后端代理化

现在虽然可以在前端配置模型 Key，但正式上线不应让前端直接持有 API Key。下一步应做后端代理：

- 前端只传用户请求和人物上下文。
- 后端保存或转发模型密钥。
- 后端做限流、日志、安全过滤和错误处理。

### 4.2 缺少真实账户和云同步

目前数据仍以本地缓存和 JSON 导入导出为主。后续需要：

- 用户注册登录。
- 云端数据库。
- 多设备同步。
- 离线模式和云同步并存。
- 加密备份。

### 4.3 日程仍偏展示型

日程页当前主要自动生成建议，还缺少：

- 手动新增日程。
- 编辑时间。
- 完成 / 延期 / 忽略。
- 与人物记忆联动。

### 4.4 回归测试还需要真实跑一遍

已经有测试清单和验证脚本，但还需要打开页面逐项跑：

- 全页面视觉检查。
- 对话流程检查。
- 导入文件检查。
- 桌面版窗口适配检查。
- 打包安装包检查。
