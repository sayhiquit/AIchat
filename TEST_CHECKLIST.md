# 回归测试清单

每次优化、同步桌面版、打包安装包前，按这份清单跑一遍。

## 1. 基础验证

- 运行 `npm.cmd run validate`，确认全部 OK。
- 打开网页版或桌面版，确认首页无白屏。
- 切换浅色 / 深色模式，确认主要页面文字可读。
- 刷新页面，确认本地数据不会异常丢失。

## 2. 人际网

- 点击分类节点，确认地图移动和分支展开正常。
- 在一个分类展开后切换另一个分类，确认旧分支不会残留。
- 点击人物节点，确认其他节点渐隐，右侧出现人物面板。
- 点击头像或返回，确认能回到中心地图。
- 在小屏宽度下，确认左侧关系事项面板可收起。

## 3. 分类和人物 CRUD

- 新增分类，确认人际网和管理页都能看到。
- 编辑分类名称和描述，确认保存后显示一致。
- 删除分类，确认该分类人物被迁移到其他分类。
- 新增人物，确认取消不会保存空人物。
- 编辑人物关系、重要程度、人物背景，确认详情页展示正确。
- 删除人物，确认二次确认弹窗正常，删除后不残留选中状态。

## 4. 人物详情和蒸馏

- 打开人物详情，确认画像、背景、记忆管理、蒸馏输入框都显示。
- 手动输入一段聊天，点击蒸馏，确认画像和历史记忆更新。
- 导入 TXT / Markdown / HTML 聊天记录，确认能识别并提示识别数量。
- 导入不可读 BAK/BAT，确认提示清晰，不崩溃。
- 编辑历史记忆并保存，确认刷新后还在。
- 删除历史记忆，确认不会影响其他人物。

## 5. 对话助手

- 选择“对方所说”，输入对方的话并发送，确认对方气泡先出现。
- 生成回复选项后，点击 A/B/C，确认内容进入底部输入框。
- 直接自定义回复并发送，确认选择气泡消失，我方气泡显示发送内容。
- 开启自动 AI 回复时，确认出现“正在输入中”动态气泡。
- 使用“我方述说”，确认我方消息先显示，再模拟对方回复。
- 修改 AI 预测的对方回复，确认能重新生成我方回复建议。
- 点击新增记忆但不保存，切换页面时应提示未保存。
- 保存记忆后，确认模拟对话才写入真实历史。
- 刷新后，未保存的模拟对话应清空，真实历史应保留。

## 6. AI 设置

- 点击 DeepSeek 预设，确认 Base URL 和模型被填入。
- 未填 API Key 时点击测试连接，应提示缺少配置。
- 填写 Key 后点击测试连接，应显示成功或明确错误。
- 关闭在线 AI 时，对话和蒸馏应回退本地规则模拟。
- 在线 AI 请求失败时，应显示回退提示，不应卡死页面。

## 7. 数据迁移

- 导出记忆 JSON，确认文件能下载。
- 新环境导入 JSON，确认分类、人物、记忆恢复。
- 导入格式错误文件，应显示失败提示，不破坏当前数据。
- 离线模式开关刷新后保持状态。
- 设置页显示当前存储位置。

## 8. 日程页

- 打开日程页，确认时间表、月历、本周维护分布正常。
- 点击人物相关日程，确认能跳转到对应人物详情。
- 人物为空时，日程页仍应有空状态或默认建议，不白屏。

## 9. 桌面版

- 运行 `npm.cmd run desktop`。
- 窗口标题、菜单中文显示正常。
- 缩小窗口到最小宽度，布局不应严重遮挡。
- 关闭并重开桌面版，本地数据不丢失。
- 打包前确认 `package.json` 版本号已更新。

## 10. 发布前检查

- `git status --short` 只包含预期改动。
- 不提交真实 API Key、真实人物库、聊天记录和安装包产物。
- GitHub 推送完成后，确认远程仓库最新提交正确。
